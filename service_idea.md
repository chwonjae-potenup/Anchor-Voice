# Anchor-Voice 서비스 아이디어

> **20대를 위한 지능형 모바일 뱅킹 에이전트**  
> CDD 기반 안면 인식과 대화형 LLM으로 편의성과 보안을 모두 잡다.

**팀:** 4조 (장원재, 김금비, 민채영)

---

## 1. 기획 배경 (Why?)

### 보이스피싱 == 어르신 피해인가?

> **"더 이상 고령층만의 문제가 아니다"**  
> **"청년층이 매우 큰 피해군으로 부상"**

- 연령대별 보이스피싱 피해 비율 (2025년 8월 기준)
  - 20대 이하: **25%**
  - 60대: **25%**
- 청년층과 고령층이 동시에 취약한 **양극화 구조** 형성
- 2022년 31% → 2023년 47% → 2024년 26% → 2025년 8월 25%

### 왜 청년층이 취약해졌는가?

- **20대 이하**는 기관사칭, **30~40대**는 대출빙자 수법에 취약
- 사회초년생인 20대 피해자의 **85.2%**가 정부·기관사칭형 사기에 당함 (금융감독원 2024.3.8)
- **원인 1:** 익숙한 모바일 환경 → 위험에 더 쉽게 노출
- **원인 2:** 사회활동 경험 부족 → 위험 상황 대응 한계

## 2. 서비스 개념 (What is Anchor-Voice?)

> **"사용자가 의심 상황에 있을 때, 대신 판단해주는 AI"**

### 핵심 기능

| 단계 | 기능 | 설명 |
|------|------|------|
| 0 | **얼굴 등록** | 본인인증용 얼굴 사진 등록 (별도 상태 탭에서 선택적 진행) |
| 1 | **CDD 위험도 산출** | 시중은행 계좌 검증(번호 길이 등) + 거래패턴으로 0~100점 스코어링 |
| 2-A | **안면 동작 챌린지 인증 (저위험)** | ① DeepFace 실사 확인 → ② 2동작 콤보 명령 → ③ MediaPipe 통과<br>*(※ 얼굴 미등록 시 예전 방식인 비밀번호/공인인증서 대체 인증 수행)* |
| 2-B | **Anchor-Voice 음성 대화 (고위험)** | TTS 질문 → 시연 PC 마이크 녹음 → STT 변환 → 화면 표시 → LLM 텍스트 분석 |
| 3 | **스텔스 SOS** | 피싱 탐지 시 위장 이체 완료 UI + 실제 차단 |


### 핵심 가치 4가지

1. **사전 예방** — 사후 대응이 아닌 피해 발생 전 차단
2. **상황 기반 판단** — 단순 정보 제공이 아닌 맥락 분석
3. **실제 음성 대화형 인터랙션** — TTS로 질문하고 음성으로 대답, LLM이 내용 자체를 분석
4. **즉각 개입** — 스스로 판단하기 어려운 그 순간에 AI가 개입

## 4. 핵심 기술 및 아키텍처

### CDD 기반 동적 라우팅 (Dynamic Routing)

```
이체 시도
    ↓
CDD(고객확인제도) 기반 위험도 판별
    ├── [저위험군] → 1초 안면 인식 (DeepFace) 패스 → UX 극대화
    └── [고위험군] → 대화 인증 (Anchor-Voice) 강제 진입 → 보안 극대화
```

### LLM 앵커 보이스 & 스텔스 SOS

- **앵커 보이스:** STT/TTS 연동, 타이핑 없는 실시간 대화 인증 및 위험도 스코어링
  - "상대방이 캡처를 요구하나요?" 등 압박 질문으로 피싱 판별
- **스텔스 SOS:** 가해자 감시 회피를 위한 **'위장 이체 완료 UI'** 렌더링 및 실시간 차단

### 기술 스택

| 분야 | 기술 |
|------|------|
| Vision | DeepFace (실사 검증 / 딥페이크 방지) + MediaPipe (동작 인식: 고개방향·눈 깜빡임) |
| Audio | Whisper STT (PC 마이크) & gTTS (PC 스피커) |
| Brain (LLM) | Gemini / GPT (피싱 판별 + 2콤보 동작 챌린지 명령 생성) |
| Backend | FastAPI |
| Frontend | Streamlit (`st.camera_input` 웹캠 / `audio_recorder_streamlit` 마이크) |


### 사기 패턴 역엔지니어링 프로세스

1. **수집** — 금감원 '그놈 목소리' 실제 피싱 오디오 확보
2. **전처리** — Whisper STT로 비정형 음성을 텍스트화
3. **분석** — 사기꾼의 심리적 압박 논리 파악
4. **도출** — '5가지 핵심 앵커 질문' 프롬프트 구축

## 5. 서비스 플로우

```
① 실시간 위험 분석
   이체 요청 시 행동 패턴 및 수취 계좌 위험도 필터링
        ↓
② 동적 인터페이스 라우팅
   점수에 따라 안면 인식 또는 음성 인증으로 유도
        ↓
③ 인공지능 개입 및 차단
   대화 분석 및 '스텔스 SOS' 위장 UI 노출
```
